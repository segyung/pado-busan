# 🌊 PADO — Netlify 배포용 완성본

## 포함 기능
- 로그인 / 회원가입
- 회원가입 직후 `승인 대기중` 상태 저장
- 승인 대기 페이지 `/pending`
- 승인 거절 페이지 `/rejected`
- 비로그인: 밋업 / 후기 / 명예의 전당 둘러보기 가능
- 승인 완료 회원: 밋업 만들기, 참가 신청, 후기 작성, 마이페이지 가능
- 크루 대시보드: 회원 승인 / 거절 / 활동정지 / 권한 변경 / 밋업 승인 / 후기 관리
- 헤더: 로그인 전 `로그인·회원가입`, 로그인 후 회원 드롭다운
- Netlify 설정 파일 포함

## Netlify 배포 방법
1. 이 ZIP 파일 압축 풀기
2. GitHub에 올리거나 Netlify에서 폴더 업로드
3. Build command: `npm run build`
4. Publish directory: `dist`

`netlify.toml`이 포함되어 있어 보통 자동 인식됩니다.

## 로컬 실행
```bash
npm install
npm run dev
```

## 크루 접속
상단 `크루` 버튼 클릭 → 비밀번호 입력

기본 크루 비밀번호:
```text
pado2026
```

## 회원 상태값
```text
pending   승인 대기
approved  승인 완료
rejected  승인 거절
suspended 활동 정지
```

## 주의
현재 버전은 빠르게 배포 가능한 프로토타입 구조입니다. Firestore 보안 규칙은 테스트용으로 열려 있습니다. 실제 운영 전에는 Firebase Auth 기반으로 보안 규칙을 강화하는 것을 추천합니다.
